# Career Coaching Survey

Psychometric survey platform for Vietnamese GenZ career guidance.
Students complete 180 questions (MBTI + Holland + OCEAN), and receive a consultant-facing report with personality and career match analysis.

## How It Works

### Survey System

Google Forms collects 180 questions across 3 tests:

| Test | Questions | Method | What it measures |
|------|-----------|--------|-----------------|
| MBTI | Câu 1–60 | Average | Personality style (E/I, S/N, T/F, J/P) |
| Holland / RIASEC | Câu 61–120 | Sum | Career interest groups (R/I/A/S/E/C) |
| OCEAN / Big Five | Câu 121–180 | Average + reverse | Personality dimensions (O/C/E/A/N) |
| SSS | Derived | Weighted composite | Social synchronization score |

### Scoring

Scoring happens in two places:

1. **Google Sheets Scores tab** — formulas generated by `deploy.py` from the survey JSON. Every new student submission automatically gets scored via `onFormSubmit` in Apps Script (which copies the seed row formulas to a new row).

2. **`scorer.py`** — Python implementation of the same scoring logic. Used for unit testing and integration testing. Acts as the ground truth to verify the Scores tab formulas are correct.

### Updating the Survey

When questions or scoring change:

1. Your survey designer provides a new 180-question set
2. Their AI assistant converts it to JSON using the prompt below
3. Drop the new file into `src/survey_versions/survey_v3.json`
4. Run `python src/deploy.py src/survey_versions/survey_v3.json`
5. Done — Google Form and Scores tab are both updated and tested

```bash
pip install google-auth google-auth-oauthlib google-api-python-client jsonschema python-dotenv pytest
```

### 2. Set up environment variables

```bash
cp .env.example .env
```

Edit `.env` and fill in:

```dotenv
GOOGLE_SERVICE_ACCOUNT_JSON=src/google_service_account.json
GOOGLE_FORM_ID=1a541_J6tBfLOoQB-ZdNH8I1UBlYw1qQ4MS6dPJvsb3c
GOOGLE_SHEET_ID=your_spreadsheet_id_here
RESPONSE_SHEET_NAME=Form Responses 2
SCORES_SHEET_NAME=Scores
```

**Finding your `GOOGLE_SHEET_ID`:**
Open your Form Responses spreadsheet → copy the ID from the URL:
```
https://docs.google.com/spreadsheets/d/THIS_PART/edit
```

### 3. Set up Google Service Account

1. Go to [console.cloud.google.com](https://console.cloud.google.com)
2. Create a project (or use existing)
3. Enable these APIs:
   - Google Forms API
   - Google Sheets API
4. Go to **IAM & Admin → Service Accounts → Create Service Account**
5. Name it `career-coaching-deploy` → Create and Continue → Done
6. Click the service account → **Keys** tab → **Add Key → JSON**
7. Save the downloaded file as `src/google_service_account.json`
8. Share your Google Sheet and Google Form with the service account email:
   - It looks like: `career-coaching-deploy@your-project.iam.gserviceaccount.com`
   - Give it **Editor** access on both

## Running Tests

### Unit tests (no Google credentials needed)

```bash
python -m pytest tests/test_scorer.py -v
```

48 tests covering:
- All 8 MBTI axis averages and gaps
- MBTI type and clarity interpretation
- All 6 Holland group sums and Top 3
- All 5 OCEAN dimension averages with reverse scoring
- SSS all 3 components + total + interpretation
- Edge cases (empty answers, all 1s, all 5s)
- Schema integrity (question count, numbering, axis consistency, weights)

### Integration test (requires Google credentials)

```bash
python src/deploy.py src/survey_versions/survey_v2.json --test
```

This injects HN-2026-0007's known answers into the live Google Sheet, reads the Scores tab output, compares it against `scorer.py` field by field, then cleans up.

## Survey JSON Format

Survey versions live in `src/survey_versions/`. Each file is validated against `src/survey_schema.json`.

### Structure

```json
{
  "version": "v2",
  "metadata": { "name": "...", "total_questions": 180, "scale": {...} },
  "tests": [
    {
      "id": "mbti",
      "name": "Test 1 — MBTI",
      "question_range": { "start": 1, "end": 60 },
      "questions": [
        {
          "number": 1,
          "axis": "E",
          "reversed": false,
          "text": "Đi đâu đông vui là em dễ lên mood thật.",
          "low_label": "Rất không đúng với mình",
          "high_label": "Rất đúng với mình"
        }
      ]
    }
  ],
  "scoring": {
    "tests": [...],
    "composite_scores": [...]
  }
}
```

### Key rules

- Total questions must equal `metadata.total_questions` (currently 180)
- Question numbers must be sequential 1–180 with no gaps
- MBTI uses `method: "average"` — bipolar axes (E vs I, S vs N, T vs F, J vs P)
- Holland uses `method: "sum"` — 10 questions per group, max 50 per group
- OCEAN uses `method: "average_with_reverse"` — some questions scored as `6 - score`
- Reversed questions must have `"reversed": true` in the question definition AND appear in the scoring group's `reversed` array
- Composite score component weights must sum to 1.0

### Prompt for generating a new survey JSON

Use this prompt with your AI assistant when a new survey version is ready:

```
Bạn là trợ lý chuyên chuyển đổi bộ câu hỏi hướng nghiệp sang định dạng JSON.

Nhiệm vụ: Chuyển toàn bộ 180 câu hỏi và cấu hình chấm điểm dưới đây sang file JSON
theo đúng schema.

SCHEMA (cấu trúc bắt buộc):
[DÁN NỘI DUNG FILE src/survey_schema.json VÀO ĐÂY]

QUY TẮC BẮT BUỘC:
1. Tổng số câu phải đúng 180 (MBTI: câu 1-60, Holland: câu 61-120, OCEAN: câu 121-180)
2. Mỗi câu phải có đủ: number, axis, reversed, text, low_label, high_label
3. Câu có đánh dấu [R] hoặc đảo chiều thì reversed = true, còn lại reversed = false
4. Holland dùng method = "sum", MBTI và OCEAN dùng method = "average" hoặc "average_with_reverse"
5. Điền đúng version (ví dụ "v3") vào field "version"
6. Không thêm bất kỳ field nào ngoài schema
7. Output chỉ là JSON thuần túy — không có giải thích, không có markdown, không có ```json

BỘ CÂU HỎI VÀ CHẤM ĐIỂM:
[DÁN TOÀN BỘ 180 CÂU VÀ HƯỚNG DẪN CHẤM ĐIỂM VÀO ĐÂY]
```

## Adding a New Composite Score

To add a new composite score (e.g. a Creative Curiosity Score), add it to the `composite_scores` array in the survey JSON:

```json
{
  "id": "ccs",
  "name": "Creative Curiosity Score",
  "label": "CCS",
  "components": [
    {
      "source": "test_group",
      "weight": 0.50,
      "test_id": "ocean",
      "group_id": "O"
    },
    {
      "source": "test_group",
      "weight": 0.50,
      "test_id": "holland",
      "group_id": "A"
    }
  ],
  "interpretation_thresholds": [
    { "max": 1.80, "label": "Thấp" },
    { "max": 3.20, "label": "Trung bình" },
    { "max": null, "label": "Cao" }
  ]
}
```

Supported `source` types:

| source | description |
|--------|-------------|
| `bipolar_ratio` | Ratio formula e.g. E/(E+I) scaled to 1–5 |
| `test_group` | Average or sum of a named group from a test |
| `question_subset` | Average of a custom set of questions with optional reverse scoring |

Then run `python src/deploy.py src/survey_versions/survey_v3.json` — the new score is automatically added to the Scores tab formulas and tested.
